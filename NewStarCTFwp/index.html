Week1 [36/36]

pwn[5/5]

ret2text

打开ida，看到main有个栈溢出，然后查看backdoor里面执行了execve，所以把main返回地址改为0x401212那里就行了

.text:0000000000401212 BA 00 00 00 00 mov edx, 0 ; envp
.text:0000000000401217 BE 00 00 00 00 mov esi, 0 ; argv
.text:000000000040121C 48 8D 05 F4 0D 00 00 lea rax, path ; “/bin/sh”
.text:0000000000401223 48 89 C7 mov rdi, rax ; path
.text:0000000000401226 E8 65 FE FF FF call _execve

ezshellcode

没有任何输入检测，直接用pwntools自带的shellcraft模块生成shellcode注入就行

newstar shop

显然是整数溢出，money应该是用无符号数储存（也提示了这一点），所以这里只要一直花钱让money小于0，就可以符号溢出，有很多钱了。

看ida的data段money被初始化为0x64，也就是100块。所以花费大于100就行了。

要注意最后一定只能通过dont_try那里-50来溢出，因为在shop那里会对你的剩余money做检测，所以shop那里是不能符号溢出的

由于开了PIE，基地址是不确定的，但是由于基地址最低3个16进制位为0，所以低3位的偏移还是可以确定的，我们发现溢出的位置是在sub120E这个函数里，main中调用了这个函数，所以sub120E原本的返回地址应该是在0x1297那里，刚好和system那里只有最低字节不同，所以可以把返回地址的最低字节覆盖为6c，就可以执行后门getshell了。

random

程序设置了srand(time(0))

由于远程那边启动和本地加载libc有一定时间差，我在远程连接后本地获取的time0返回值减了1，然后就rand（）获取随机数发送就行了。

只能说还是有一定运气成分的，有时候不一定能随机到$0，我是大概试个两三次就可以了。

咳

upx -d 脱壳，ida启动！发现就是把输入的每个字符加1，所以直接把enc里面的数据（ida里面双击enc查看）每个-1就行了。

Segments

打开ida然后shift+f7查看段，发现了开头的flag_和结尾处的name_， misc手的直觉告诉我，这是flag{和name}

所以把那些奇怪的段名拼接起来就行了

flag{You_ar3_g0od_at_f1nding_ELF_segments_name}

ELF

先经过一个出题人自定义的encode函数，在来一个经典版base64

记得解base64用bytes不要用str，一眼丁真base64会解出来一堆不可见字符

Endian

就是简单的异或，小端序可以直接用p32包一下就行了。

解出来没有最后的}，要自己加上去

AndroXor

直接jadx反编译，（就贴了主要逻辑）

直接复制过来，注意python不像java这些类C语言是可以字符当一个i8或u8来运算，所以还要手动转一下

EzPE

用010editor或者其他的16进制编辑器改PE头

文件头是应该是4D 5A 90 00， 也就是说前面应该是MZ..，把W改成M；

然后后面的0x3c开始的几个字符要指向一段PE开头的东西（50 45 00 00 …..)，可以看到是在0x80的位置，把这里的90改成80就可以了。

然后打开ida开逆！

查看enc选中enc的所有数据，按shift+E来提取数据，直接ctrl+C和ctrl+V复制效果不太好

然后直接在资源搜索这个字符串，然后在res/layout/layout_2.xml中找到了这个资源，flag就在里面

Affine

发现还是要求key，这次有两个。很容易可以把key的范围缩小到0到255，所以可以直接遍历2层256的循环，然后还是利用前4个字符是“flag”，来求key[0], key[1]。 但是求了只会呢。这里要用到除法取模等价于模逆元取模。

babyAES

这里os.urandom（16）返回一个16字节的bytes，乘2就是两个一样的拼在一起，可以利用这个先取高16字节来确定key，然后异或低16字节求得iv，就可以解aes了

Begin of PHP

level 1是md5碰撞，利用php会把0e解释为科学计数法（如果后面全是数字的话）。也就是0。所以要找2个字符串md5后是0e开头，而且后面全是数字的。（网上随便找的到的）

level 2原理类似，变成sha1和md5了，太难找了，换了一个思路，利用传入数组会返回null的特性，也就是null==null绕过检测

level 3利用strcmp传入数组会返回0来绕过

level 4可以在传入的数后面加一个%20或者%00，这样可以绕过is_numeric，同时也不会影响后面和数的比较

level 5利用extract可以解析并创建变量，来psost一个flag5进去（因为发现源码中没有flag5）。另外对post内容做了限制，不能有字母数字，这里可以传一个![]进去，[]空数组会隐式转换为false，在！取反得到我们要的true

R!C!E!

爆破试出来114514的md5前6位符合要求，第二个post数据一开始post到e_v.a.l不行，因为php中变量名不能有点，会把点转化为下划线。导致实际上post到e_v_a_l了。但是实际上我们是需要有点的。后来发现变量名解析[符合会把它转化为下划线，然后不会把后面的点解析为下划线了。所以可以post的变量名可以填e[v.a.l

然后还要绕过一个正则，这里直接用点来拼接出一个flag字符串就行了

EasyLogin

注册登陆发现有一个302的请求，发现里面有一个假的flag，由于有一个admin账户，猜测登陆admin的302请求会有flag。

先尝试burp爆6位的纯数字，发现有一个请求长度明显比其他的长

因为发送的password是经过md5加密的，把那一段670开头的东西解密出来得到密码000000，回去试着登录，成功登录，然后burp抓包看那个302的包，flag就在注释里。
